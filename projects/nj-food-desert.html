NJ Food Environment ML Pipeline

Predicting food desert status across New Jersey zip codes using machine learning, Census demographics, OpenStreetMap store data, and CDC health outcomes.

Python scikit-learn GeoPandas OpenStreetMap Census ACS 2022 CDC PLACES USDA LILA Public Health
---

Overview

This pipeline builds a complete food access analysis for all NJ zip codes — from raw OpenStreetMap queries and Census API calls through spatial joins, ML classification, and downstream health outcome regressions. No proprietary data or API keys are required.

The central question: can demographic data alone predict whether a community lacks meaningful supermarket access? The answer is yes, with meaningful caveats about what "desert" means depending on how you draw the boundary.

Pipeline Architecture

The pipeline runs in a single script, top to bottom, with JSON caching at every API call so re-runs skip network requests. Steps:

  1. OSM Fetch & Dedup — Overpass query for all food locations in NJ; proximity-based deduplication within 100m
  2. Spatial Join — Point-in-polygon assignment of each store to a ZCTA
  3. Proximity Features — Zip centroid → nearest supermarket distance; count within 5-mile buffer
  4. Census ACS — Two-batch API pull; feature engineering (poverty rate, SNAP %, no-vehicle %, elderly share, etc.)
  5. Target Construction — Two targets: zip-level binary (any supermarket in zip) and 5-mile buffer binary (USDA-aligned)
  6. ML Training — Logistic Regression, Random Forest, Gradient Boosting; stratified split; class-balanced training
  7. Validation — Bootstrapped CIs, leave-one-county-out spatial CV, USDA LILA agreement (Cohen's κ)
  8. Health Regressions — OLS: desert probability → CDC PLACES outcomes, controlling for PCA deprivation index
  9. Typology — Six food access categories: Desert, Fragile Access, Urban Mirage, Rural Mirage, Swamp, Healthy Access

Food Access Typology

Beyond a binary desert flag, each zip code is assigned to one of six access categories based on supermarket count, 5-mile buffer access, food swamp ratio, population density, and demographic vulnerability:

CategoryDefinition
DesertNo supermarket in zip or within 5 miles
Urban MirageStore present but demand far exceeds capacity (top decile of mirage score)
Rural MirageLow-density zip with limited within-5mi options and high distance burden
SwampUnhealthy options (fast food + convenience) outnumber supermarkets >1:1
Demographic FragilityHigh elderly share, low transit access, sparse nearby stores
Fragile AccessTechnically served but thin coverage (≤1 store in zip or ≤2 within 5mi)
Healthy AccessAll other zip codes

Spatial Validation

Leave-one-county-out cross-validation was used alongside standard k-fold to test whether the model generalizes across NJ's 21 counties or merely memorizes within-county patterns. Agreement with the USDA LILA classification was measured using Cohen's κ.

Disagreements between the model and USDA are structurally meaningful: the model uses OSM store presence and a 5-mile buffer, while USDA applies an income gate at the census tract level. Zips that appear in one but not the other typically sit at the urban-rural fringe or near county lines.

Health Outcome Regressions

OLS regressions linked desert probability to CDC PLACES health outcomes at the zip code level, controlling for a PCA-derived economic deprivation index (PC1 of poverty rate + median income). This removes the collinearity that plagues direct poverty + income controls.

Depression, coronary heart disease, and high blood pressure showed the strongest positive associations with desert probability. Food swamp ratio (unhealthy options per supermarket) was most strongly correlated with tooth loss — a known marker of diet quality and healthcare access — and was the dominant predictor over desert status for several outcomes.

Vulnerability Analysis

A separate query identified zip codes more than 1 mile from any supermarket with high elderly population share and high rates of households without a vehicle — the population most structurally exposed to food access gaps regardless of store presence nearby.

Technical Notes

  • OSM data deduplicated by name + type within 100m radius to remove chain duplicates
  • Proximity features excluded from model inputs — used only to define the target variable (no leakage)
  • Gradient Boosting trained with per-sample weights to handle ~7.6:1 class imbalance in the 5-mile target
  • Threshold tuned via spatial CV F1 rather than defaulting to 0.5
  • Census suppressed sentinel values (<0) replaced with NaN; median_rent and median_home_value excluded due to high missingness in small ZCTAs
  • PCA deprivation index avoids multicollinearity between poverty rate and median income in regression controls
